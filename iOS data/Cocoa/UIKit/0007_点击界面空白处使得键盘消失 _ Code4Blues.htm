点击界面空白处使得键盘消失

在输入内容之后，除了点击Return可以使得键盘消失之外，点击界面空白处让键盘消失也是非常方便的。

使用UIGestureRecognizer

一种方法是使用UIGestureRecognizer来实现，同时利用了UIGestureRecognizerDelegate中的-(BOOL)gestureRecognizer:shouldReceiveTouch:方法。

这种方法首先要让界面的ViewController ConfirmUIGesutureRecognizerDelegate，然后在界面初始化的时候添加一个UITapGestureRecognizer。在相应的Action中对self.view发送endEditing:消息：

- (id)initWithNibName:(NSString *)nibNameOrNil bundle:(NSBundle *)nibBundleOrNil {
  self = [super initWithNibName:nibNameOrNil bundle:nibBundleOrNil];

  if (self) {
      UITapGestureRecognizer *backgroundTap = [[UITapGestureRecognizer alloc] initWithTarget:self action:@selector(backgroundTapped:)];
      backgroundTap.delegate = self;
      [self.view addGestureRecognizer:backgroundTap];
  }
}

- (void)backgroundTapped:(id)sender {
  [self.view endEditing:YES];
}

一般情况下这样就可以保证键盘在我们点击界面其他地方的时候会消失。但是这会造成一个问题，如果我们在界面的下方有一个UIToolBar的时候，UIToolbar中的按钮（UIBarButtonItem）是不会响应其动作的。

至于为什么没有响应，个人感觉是UIBarButtonItem虽然是被点击了，但是其响应得却是我们前面注册的手势的回调。

具体是Responder Chain还是什么的原因，这里就不探究了。可以是说手势动作的优先级高于了按钮本身动作的优先级，这显然是个Bug（这个Bug是在iOS 7上出现的，低版本没试过，但是在iOS 8中被干掉了）。

这时候我们就需要辨别Tap动作的作用对象，上面没有用到的UIGestureRecognizerDelegate就起到作用了。

实现上面提到的-(BOOL)gestureRecognizer:shouldReceiveTouch:方法，我们需要在该方法中判断touch.view是不是工具栏按钮本身的View。

如果我们在方法中加上断点观察点击到工具栏按钮时，touch.view的class，可以看到其类是UIToolbarButton，而这个类不是公开，我们是没有办法使用到它。只能退而求其次，判断touch.view是否是UIControl了。而为什么可以使用UIControl呢，因为UIToolbarButton的superclass就是UIControl。而对于UIControl来说，如果它能响应用户交互事件，则会自动响应本身的事件，而不是我们的手势事件。

代码如下：

- (BOOL)gestureRecognizer:(UIGestureRecognizer *)gestureRegonizer shouldReceiveTouch:(UITouch *)touch {
  if ([touch.view isKindOf:[UIControl class]]) {
    return NO;
  }
  return YES;
}

这样，在iOS 7中，就能使得键盘在点击界面背景时消失了，副作用不大。。。

使用UIControl

UIControl是UIView的子类，如果界面的UIView也能像按钮那样设置Action，可以直接点击界面触发相关动作使得键盘消失，那就方便得多了。

可惜是没有办法对直接加上Action的，要动用IB。选中界面所在的xib文件，然后选中view，在Utilities面板中选中Identity Inspector，可以看到Custom Section下面是Class一项的值是UIView，我们把它改成UIControl。

接着打开Assistant Editor，按住option选中界面的实现，然后从view上面Control-drag到实现，这时就可以看到平时设置按钮动作一样的弹出框，Event类型为Touch Up Inside，在生成的IBAction方法中跟前面一样发送endEditing:方法就行了。

这个法子比前面Gesture的方法要简单很多...
